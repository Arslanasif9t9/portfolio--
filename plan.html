🔍 Sab se important technical finding

arslanai9t9 mein Supabase Edge Functions NAHI hain. Uska saara AI logic (provider fallback, message save, system prompt) TanStack Start ke createServerFn server functions mein hai — ye sirf uske apne Node/SSR server par chalta hai.

Portfolio ek static site hai (GitHub Pages) — iska koi server nahi. Isliye:

  • AI logic ko Supabase Edge Function (Deno) mein rewrite karna hoga → supabase/functions/chat/
  • API keys (Gemini/Groq waghera) browser mein kabhi nahi aayengi — wo DB (api_providers table) mein rehti hain aur sirf Edge Function unhe padhta hai. ✅ Secure.
  • Chat UI ka React code seedha port ho jayega (React 19 → 18 koi masla nahi, standard hooks hi use hue hain).

Decision: Same Supabase project reuse karenge

arslanai9t9 wala Supabase project (ifdnphgjifdtefngmnha) hi use hoga kyunki:

  • api_providers mein aapki AI keys pehle se configured hain — dobara setup nahi.
  • Portfolio ki chats bhi aapke existing admin panel (/admin9t9) mein nazar aayengi — ek hi jagah sab manage.
  • Tables (chat_sessions, messages, RLS policies) already anonymous guests support karte hain.
  • Sirf 1 nayi table add hogi: contact_messages (contact form ke liye).

🔐 Security Notes

  • ⚠️ arslanai9t9 repo mein .env committed hai (URL + publishable key). Publishable key itni sensitive nahi, lekin best practice: GitHub par public hai to keys rotate kar lein. Portfolio mein .env kabhi commit nahi hoga.
  • Service role key sirf Edge Function secrets mein — code/browser mein kabhi nahi.
  • AI provider keys DB mein hain, sirf server-side read hoti hain — browser ko sirf reply milta hai.
  • Contact form: anon sirf INSERT kar sakta hai, kisi ka data read nahi.
